# hiro-lab Site Builder Checklist

## Purpose
このチェックリストは、**作る行為自体が美学を再生産する**ための最小基準です。
ページ追加・変更・修正の前後で必ず回してください。

## 0. 作業開始前
- [ ] 変更対象が `site/templates/` にある場合は、テンプレを変更しない
- [ ] 新規ページを作る場合、**最初に templates/ の母型をコピー**している
- [ ] 変更理由を 1 行で書ける（セクション追加 / 文言修正 / 等）

## 1. 色と質感（和紙 × 濃墨）
- [ ] body 背景: `--color-bg` (#f5f1e8) をそのまま使っている
- [ ] カード・セクション: `--color-surface` / `--color-border` を使い分けている
- [ ] 文字: `--color-text` (#1a1a1a) / 墨グレー `--color-text-muted` のみ使用
- [ ] 朱色 (vermilion) は強調点にだけ使い、全体に撒いていない
- [ ] 和紙ノイズ (`body` の data URI) を消していない

## 2. タイポグラフィ（明朝中心）
- [ ] 本文・見出しに明朝 (`--font-mincho`) を適用している
- [ ] 見出し breaks を入れすぎていない
- [ ] Letter-spacing は 0.08em〜0.18em の範囲で固まっている
- [ ] 明朝 weight は 400〜500。700 等は使っていない
- [ ] 行送り (line-height) は 1.75〜1.95 の範囲

## 3. 時間と余白
- [ ] Section 余白は 4rem 前後 / カード余白は 1.25rem 前後で安定
- [ ] カードと見出しの上下マージンが“同じ数値”で揃っている
- [ ] モバイルで明朝見出しが折り返さないか、clamp で制御している

## 4. 階層と視線誘導
- [ ] H1 → H2 → 本文 のサイズ/weight 差が明確
- [ ] カテゴリタグ > タイトル > 抜粋 > 日付 の優先順が視覚的に成立
- [ ] CTA ボタンが“押せる面積”を持っている（幅 / padding が他より大きい）
- [ ] “何もない余白” が意図的に残されている

## 5. 共通コンポーネント
- [ ] 記事カード: 形・影・境界線のロジックを変えていない
- [ ] ランキング: 順位 circles (nth-child 1/2/3) の朱・銀墨・銅墨を保持
- [ ] タグ: pill 形状 / hover 反転の挙動を遵守
- [ ] 検索モーダル: backdrop + panel の構造を変えていない

## 6. アクセシビリティ / パフォーマンス
- [ ] `prefers-reduced-motion` で墨演出が停止する
- [ ] キーボード操作で検索モーダルが開く (Cmd/Ctrl+K)
- [ ] モバイルでナビが開閉する (nav-toggle)
- [ ] 画像 alt / OGP / JSON-LD を削除していない

## 7. 確認
- [ ] `http://localhost:8765/` で最終確認した
- [ ] 追加したページが前列のページと“同じ空気”を出している
- [ ] 違和感があれば、 Checklist の該当項目を先に直す
